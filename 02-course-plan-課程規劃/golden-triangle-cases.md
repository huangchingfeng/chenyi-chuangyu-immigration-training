## 1. 專業文案撰寫

### 案例 1-1：EB1A 推薦信（Tyng 專用）

**情境**：客戶是科技業 CTO，需要撰寫 EB1A 推薦信

```
【角色 Role】
你是一位擁有 15 年經驗的美國移民律師，專精 EB1A 傑出人才申請案件，
曾協助超過 200 位科技業高階主管成功取得綠卡。

【情境 Context】
我需要為一位科技公司 CTO 撰寫 EB1A 推薦信。
- 客戶背景：AI 新創公司 CTO，擁有 5 項美國專利
- 推薦人：前 Google 資深工程師（客戶的前主管）
- 目的：強調客戶在 AI 領域的傑出貢獻與原創性
- 使用場景：作為 EB1A 申請文件的一部分提交給 USCIS

【期望 Expectation】
請協助撰寫一封專業的 EB1A 推薦信，需符合以下要求：
1. 字數：800-1000 字（英文）
2. 結構：開場介紹推薦人資格 → 與申請人的關係 → 具體成就描述 → 對領域的影響 → 結論
3. 語氣：專業、客觀、有說服力，避免過度誇張
4. 重點：強調「原創性」（Originality）和「對領域的重大貢獻」（Major Contribution）
5. 格式：正式商業信件格式，包含推薦人簽名區
```

**教學重點**：
- ✅ 角色要具體（15 年經驗、200+ 案件）
- ✅ 情境要完整（客戶背景 + 推薦人 + 目的 + 使用場景）
- ✅ 期望要明確（字數、結構、語氣、重點、格式）

### 案例 1-2：NIW 申請信（Tyng 專用）

**情境**：客戶是醫學研究員，需要撰寫 NIW 申請信

```
【角色 Role】
你是美國移民法專家，專精 NIW（國家利益豁免）申請案件，
熟悉 USCIS 的 Dhanasar 三大審查標準。

【情境 Context】
我需要為一位癌症研究員撰寫 NIW 申請信（Petition Letter）。
- 客戶背景：台灣大學醫學院副教授，專研癌症免疫療法
- 研究成果：發表 15 篇 SCI 論文，被引用超過 500 次
- 目的：證明其研究符合美國國家利益
- 審查重點：符合 Dhanasar 三大標準（重要性、定位、國家利益）

【期望 Expectation】
請協助撰寫 NIW 申請信主體內容，需包含：
1. 字數：1500-2000 字（英文）
2. 結構：
   - Prong 1：研究領域的重要性（Substantial Merit & National Importance）
   - Prong 2：申請人的獨特定位（Well-Positioned to Advance）
   - Prong 3：豁免勞工證的國家利益（Balance Test）
3. 語氣：正式、學術、說服力強
4. 引用：至少 3 個具體數據或研究成果佐證
5. 避免：空泛的形容詞，需用具體事實說話
```

### 案例 1-3：拒絕信回應（Tyng 專用）

**情境**：客戶收到 USCIS 的 RFE（補件通知），需要回應

```
【角色 Role】
你是經驗豐富的移民律師助理，專門處理 USCIS RFE（Request for Evidence）回應文件，
熟悉如何精準回應移民官的質疑。

【情境 Context】
客戶的 EB1A 申請收到 RFE，移民官質疑：
1. 推薦信內容過於主觀，缺乏具體數據
2. 媒體報導不足以證明「全國性或國際性聲譽」
3. 專利數量雖多，但未證明「原創性貢獻」

我需要撰寫一份回應信，逐一回應移民官的質疑。

【期望 Expectation】
請協助撰寫 RFE 回應信，需符合以下要求：
1. 結構：針對每個質疑點，分段回應（Issue 1, 2, 3）
2. 內容：
   - 先重述移民官的質疑（確保理解正確）
   - 提供具體證據回應（數據、新文件、專家意見）
   - 引用相關法條或判例（如適用）
3. 語氣：尊重、專業、不卑不亢
4. 字數：每個質疑點 300-500 字
5. 附件：建議需要補充哪些新文件（列清單）
```

## 4. 客戶研究

### 案例 4-1：客戶背景深度調查

**情境**：準備 EB1A 申請前的客戶研究

```
【角色 Role】
你是專業的客戶研究分析師，擅長從公開資訊中挖掘客戶的成就與資格，
協助移民顧問評估 EB1A 申請可行性。

【情境 Context】
新客戶來諮詢 EB1A，提供了以下基本資訊：
- 姓名：Dr. Chang（張博士）
- 職位：某生技公司研發總監
- 學歷：台大醫學院博士
- 自述：有發表論文、擔任期刊審稿人、獲得專利

我需要進行深度背景調查，找出所有可用於 EB1A 申請的成就證據。

【期望 Expectation】
請協助我進行客戶背景研究，並產出結構化報告：
1. 研究範圍：
   - Google Scholar：論文發表、引用次數
   - LinkedIn：職涯歷程、推薦人選
   - 公司官網：職位描述、專案成就
   - 專利資料庫：專利數量與內容
   - 媒體報導：是否有新聞報導或採訪
2. 報告結構：
   - 學術成就（論文、引用、h-index）
   - 專業貢獻（專利、產品、創新）
   - 認可與獎項（獎項、頭銜、會員資格）
   - 審查與評審（期刊審稿、評委經驗）
   - 媒體曝光（報導、採訪、演講）
3. 分析：評估是否符合 EB1A 十項標準中的至少三項
4. 建議：如果資格不足，建議客戶補強哪些部分
5. 格式：用表格整理，方便快速檢視
```
